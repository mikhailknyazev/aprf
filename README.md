# APRF article companion repository

## Key References

Curated LinkedIn articles by **Michael Knyazev** on engineering resilient application platforms.

### Core Framework

**[Application Platform Resilience Framework](https://www.linkedin.com/pulse/application-platform-resilience-framework-michael-knyazev-phd-dssfc/)**  
*Published: February 24, 2026*  
Defines the eight disciplines of engineered resilience: Capacity, Overcommitment, Failure Domains, Lifecycle, Disruption, Hardened Applications, Guarded Chaos, and Governance & Observability.

### Practical Implementation

**[Engineering Full Application Platform Resilience with Quarkus, Java, GitOps, Ansible, and OpenShift](https://www.linkedin.com/pulse/engineering-full-application-platform-resilience-java-knyazev-phd-ha1lc/)**  
*Published: April 29, 2026*  
A hands-on reference implementation on Azure Red Hat OpenShift (ARO) demonstrating the framework in action with Quarkus (JVM + Native), real failure injection, GitOps corrective actions, and measurable scorecards across all eight disciplines.

## What is this repository for?

This repository publishes the companion material that adds value after the
reader has already read the 2nd article in the series - "Engineering Full Application Platform Resilience with Quarkus, Java, GitOps, Ansible, and OpenShift":

- the applications/workloads source
- the simplified workload and GitOps manifests
- a small set of simplified, explanatory deploy roles

It does not publish the article manuscript itself, the broader APRF runtime, or
the raw internal APRF execution tree.
